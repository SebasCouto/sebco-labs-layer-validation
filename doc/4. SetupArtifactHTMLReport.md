# Configurar un reporte HTML local y remoto

Para configurar el reporte necesitamos realizar los siguientes pasos:

## 1. Agregar la depencia en el requirements.txt
La dependencia `pytest-html` en el archivo `requirements.txt` indica que tu proyecto de Python utiliza un complemento para pytest que genera informes HTML detallados después de ejecutar las pruebas. Al incluir esta dependencia, se habilita la capacidad de generar informes visuales más legibles y accesibles sobre el resultado de tus pruebas.

Desde el file requirements.txt agregar el siguiente comando:
```sh
pytest-html
```
La dependencia pytest-html y su configuración en el archivo pytest.ini son importantes si deseamos generar informes detallados en formato HTML después de ejecutar las pruebas con pytest. Este complemento proporciona una representación visual más amigable de los resultados de las pruebas, lo que facilita la identificación y resolución de problemas en el código.

## 2. Cofigurar en el archivo pytest.ini el addopts para alojar el reporte en HTML
Este archivo se utiliza para configurar opciones y ajustes específicos al ejecutar pruebas con pytest, un marco de prueba para Python
```sh
[pytest]
addopts = -p no:warnings -s -r xX -v --tb=short --html=./report/html-report.html --self-contained-html
```
&darr;
Aquí está un detalle de las configuraciones anteriores:

- `[pytest]` Esta sección indica que las siguientes configuraciones son específicas de pytest.

- `addopts` es una opción que te permite agregar opciones de línea de comandos directamente en el archivo de configuración para evitar tener que escribirlas cada vez que ejecutas tus pruebas.

- `-p no:warnings`: Desactiva la visualización de advertencias durante la ejecución de las pruebas.

- `-s`: Permite mostrar la salida de la impresión estándar (stdout) durante la ejecución de las pruebas. Esto puede ser útil para depurar o ver información de prueba.

- `-r xX`: Especifica el nivel de reporte que deseas. En este caso, se establece en "xX", lo que significa que se mostrarán solo las estadísticas finales y los casos de fallos, y se ocultarán los detalles de las pruebas que han pasado.

- `-v`: Aumenta el nivel de verbosidad. En este caso, se establece en "v", lo que significa que se mostrarán detalles adicionales sobre las pruebas que se están ejecutando.

- `--tb=short`: Controla el formato de los rastreos de pila mostrados en caso de errores. En este caso, se establece en "short", lo que significa un formato más conciso.

- `--html=./report/html-report.html`: Genera un informe en formato HTML de la ejecución de las pruebas y lo guarda en el directorio especificado.

- `--self-contained-html`: Indica que el informe HTML generado debe ser autosuficiente, es decir, debe contener todos los recursos necesarios para su visualización sin depender de recursos externos.

En resumen, este archivo pytest.ini configura diversas opciones para personalizar la ejecución y generación de informes de las pruebas realizadas con pytest. Estas opciones pueden ayudar a controlar la salida, la visibilidad de los informes y el formato de los rastreos de pila, entre otros aspectos.

## 3. Agregar en el archivo .gitignore el siguiente exclusión para no subir innecesariamente el archivo a la versión remota:

Agregar la línea `report/` en el archivo `.gitignore` es una forma de especificar que el directorio llamado "report" y todos sus contenidos no deben ser rastreados por Git ni subidos al repositorio remoto. El archivo .gitignore se utiliza para indicar a Git qué archivos o directorios debe ignorar y no incluir en el control de versiones.

```sh
report/
```
## 4. Agregar el step #7 en el ci-cd.yml para que esta configuración sea manipulable también en la corrida remota
Este step en el workflow de trabajo ci-cd.yml se encarga de generar la subida del (archivos o directorios) como en este caso el reporte HTML previamente configurado de la ejecución remota del workflow ejecutada en el repositorio de GitHub.

```sh
    steps:

      # Pasos anteriores ...

      # Paso 7: Generar un reporte HTML con la libreria artifact
      - name: Generating html report artifact
        if: always()
        uses: actions/upload-artifact@v3
        with:
          name: test-report
          path: tests/report
```
&darr;
Aquí algunas especificaciones del step:

- `steps:`: Especifica una lista de pasos que se ejecutarán en el flujo de trabajo.

- `- name`: Generating html report artifact: Define un paso en el flujo de trabajo con un nombre descriptivo. En este caso, el paso se llama "Generating html report artifact".

- `if`: always(): Esta condición if indica que este paso debe ejecutarse siempre, independientemente del resultado de los pasos anteriores. En otras palabras, este paso se ejecutará incluso si hay fallos en pasos anteriores.

- `uses`: actions/upload-artifact@v3: Utiliza una acción predefinida llamada upload-artifact de la versión 3. Esta acción se encarga de subir artefactos (archivos o directorios) como parte del flujo de trabajo, para que puedan ser utilizados en pasos posteriores o visualizados en GitHub.

- `with:`: Especifica los parámetros para la acción upload-artifact.

- `name`: test-report: Asigna un nombre al artefacto que se está subiendo. En este caso, se llama "test-report".

- `path`: tests/report: Indica la ruta del directorio (o archivo) que se subirá como artefacto. En este caso, el directorio "report" dentro del directorio "tests".
