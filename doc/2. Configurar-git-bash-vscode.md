# Configurar Git Bash como terminal por defecto en Visual Studio Code en Windows

Esta guía explica paso a paso cómo configurar **Git Bash** como la terminal predeterminada en **Visual Studio Code (VS Code)** en un sistema **Windows**, utilizando **Chocolatey** para instalar Git Bash.

---

## Requisitos previos

* Tener instalada una versión de **Visual Studio Code** igual o superior a la **1.40** (necesaria para la configuración vía `settings.json`).
  Podemos verificar la versión ejecutando este comando en la terminal (CMD o PowerShell):

  ```cmd
  code --version
  ```

* Sistema operativo **Windows** (compatible con Git Bash y Chocolatey).

---

## Pasos para configurar Git Bash como terminal predeterminada en VS Code

### 1. Instalar Chocolatey (si aún no lo tenés)

Abrimos **PowerShell** como **Administrador** y ejecutamos este comando:

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; `
[System.Net.ServicePointManager]::SecurityProtocol = `
[System.Net.ServicePointManager]::SecurityProtocol -bor 3072; `
iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
```

Esto instalará **Chocolatey**, una herramienta que permite instalar programas desde la línea de comandos.

---

### 2. Instalar Git Bash usando Chocolatey

Una vez instalado Chocolatey, ejecutamos este comando para instalar **Git Bash**:

```powershell
choco install git
```

Por defecto, se instalará en una ruta como esta:

```plaintext
E:\Program Files\Git\
```

---

### 3. Configurar Git Bash como terminal por defecto en VS Code

1. Abrimos **Visual Studio Code**.

2. Accedemos a la configuración desde `File > Preferences > Settings`, o con el atajo `Ctrl + ,`.

3. En la parte superior derecha del panel de configuración, hacé clic en el ícono de `{}` que dice **Preferences: Open User Settings (JSON)**.

4. En el archivo `settings.json`, agregamos esta configuración para usar Git Bash como terminal por defecto:

```json
"terminal.integrated.defaultProfile.windows": "Git Bash",
"terminal.integrated.profiles.windows": {
    "Git Bash": {
        "path": "E:\\Program Files\\Git\\bin\\bash.exe",
        "args": []
    }
}
```

> ⚠️ **Importante:** asegurarse de que la ruta del archivo `bash.exe` sea la correcta según su instalación. En este ejemplo es `E:\Program Files\Git\bin\bash.exe`.

---

### 4. Verificar que Git Bash esté instalado correctamente

Podés confirmar la instalación y ruta de Git con este comando:

```cmd
where git
```

Deberías ver una salida similar a:

```plaintext
E:\Program Files\Git\cmd\git.exe
```

---

### 5. Reiniciar VS Code

Después de modificar el archivo `settings.json`, cerramos y volvemos a abrir VS Code para que los cambios tengan efecto.

---

## Problemas comunes

* **No aparece Git Bash como opción en el terminal de VS Code**:
  Verificá que Git Bash esté instalado y que la ruta al `bash.exe` sea correcta.

* **Compatibilidad de versión**:
  Esta configuración funciona a partir de la versión **1.40** de VS Code. Si no ves el efecto del cambio, asegurate de tener esa versión o una más reciente.

---

## Conclusión

Con estos pasos, ya configuramos **Git Bash** como la terminal por defecto en **Visual Studio Code** en un entorno **Windows**. Esto nos permitirá trabajar con una terminal similar a Unix directamente dentro del editor.
